RUNNING

The following line contains a typical command that starts Mongoose run to create 100,000 data items of 8KB size:

java -Dstorage.addrs=10.64.84.<X>,10.64.84.<Y>,10.64.84.<Z> -Dload.limit.count=100000 -Ditem.data.size=8KB -Dauth.id=wuser1@SANITY.LOCAL -Dauth.secret=A5JKVKuSHp5Kme2qcMFlvMqEKbN+QBNF0tRuFleT -jar <MONGOOSE_DIR>/mongoose.jar
Where:
  • storage.addrs - the list of storage node IP addresses
  • load.limit.count - the number of data items to create
  • item.data.size - the size data items to create
  • auth.id - "user" authentication identifier
  • auth.secret - "user" authentication secret key

The command line above may look too long. The configuration parameters that change rarely can be stored in JAVA_TOOL_OPTIONS, which is a predefined environment variable used by Java.

Linux example:

export JAVA_TOOL_OPTIONS="-Dstorage.addrs=10.64.84.<X>,10.64.84.<Y>,10.64.84.<Z> -Dload.limit.count=100000 -Ditem.data.size=8KB -Dauth.id=wuser1@SANITY.LOCAL -Dauth.secret=A5JKVKuSHp5Kme2qcMFlvMqEKbN+QBNF0tRuFleT"

Windows example:

set JAVA_TOOL_OPTIONS=-Dstorage.addrs=10.64.84.<X>,10.64.84.<Y>,10.64.84.<Z> -Dload.limit.count=100000 -Ditem.data.size=8KB -Dauth.id=wuser1@SANITY.LOCAL -Dauth.secret=A5JKVKuSHp5Kme2qcMFlvMqEKbN+QBNF0tRuFleT

Note:
  • If no configuration properties are set in the command line or by the environment variable, default configuration is used from <MONGOOSE_DIR>/conf directory.
  • If you're running against ECS, use its provision environment to obtain credentials and other information like bucket, subtenant, container, etc.
  • Please keep in mind that high load require a lot of system resources such as open file descriptors and Java heap memory. It may be necessary to specify the resource limits manually:
    ulimit -n 1048576

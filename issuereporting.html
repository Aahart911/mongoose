ISSUE REPORTING
  1. Go to GitHub Issues and create a ticket
  2. If the issue is a bug make sure that the ticket contains the following essential information:
    • Command line used to run
    • Mode used (wsmock/standalone/webui/…)
    • Scenario used
    • Load limitations (none/count/time/manual interruption)
    • Storage API type used (Atmos/S3/Swift)
    • Storage type (mock/viper/swift native/…)
    • Conventional things: expected + observed
